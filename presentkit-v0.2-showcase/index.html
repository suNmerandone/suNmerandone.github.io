Section I · Problem

02 · 三道摩擦

resume-profile skill 要跨三道門檻

Before · 用 skill
  • 裝 Node · 裝 Claude Code · 登入
  • 讀懂 .claude/skills/ 結構
  • 手動編輯 Markdown 當輸入
  • 中途斷了 session 就沒
After · 用 Presentkit
  • 雙擊 Presentkit.App.exe
  • wizard 表單 · 自動存草稿
  • 貼 API key · 看到 LIVE 徽章才花錢
  • 按 Generate · 可取消 · 可 iterate

前者是「讓 AI 能產」的工具；後者是能被非開發者採用的產品。

Section III · The 8-step wizard

走一次
使用者的視角

Design · LinkedIn · Intake · Materials · Anchor · Outline · Generate · Done
05 · Step 1
Design

三選一，或帶自己的

  • Tech-forward Coral（預設） · 軟體 / AI / 產品
  • Corporate Neutral · 金融 / 傳產，深藍零裝飾
  • Minimalist Mono · 設計 / 編輯，全 IBM Plex Mono
或 Import custom design (JSON) 帶自己的 tokens —— accent、字型 stack 都可覆寫。
06 · Step 2
LinkedIn · optional

拖 PDF 進來 · 100% 本機

  • PdfPig 抽文字 → regex parser 拆欄位
  • Intake 空的話靜默 auto-fill
  • Intake 有內容 → 先問「覆寫嗎？」
  • 不花 token · 純本機解析
這步刻意在 Intake 之前。一份 PDF 把 7 section 表單自動填好，使用者只需校對。
07 · Step 3

Intake · 七個 section，不多也不少

1 · 姓名+Headline
一句話定位 · 決定整份履歷語氣
2 · 聯絡方式
free text，email/phone/LinkedIn/個站每行一個
3 · Target role
整個 wizard 最重要的一欄 · 決定內容選擇
4 · 現職
Company · Role · Period · bullets（有數字給數字）
5 · 過去經歷
repeater · reverse-chronological · 10+ 年舊可合併
6 · 教育
學校 · 系所 · 年份
7 · 重點成就
3–5 條 reader 8 秒掃描後會記住的

Output language 從 Intake 搬到 Generate step —— 因為 2× 成本的決策該發生在「花錢那一刻」。

08 · Step 4
Materials · optional

補充素材 · .pdf / .md / .txt

  • 職缺 JD（Claude 可以順著寫 tone）
  • 舊履歷 / cover letter 草稿
  • 專案 README · autobiographical note
這步不花 token，檔案會在 Generate 時拼進 prompt（~ 幾千 tokens 多一點成本，幾美分）。
09 · Step 5
Anchor · optional

Mirror · 或 Iterate

  • Mirror style · 承襲 voice，內容重生
  • Iterate (diff) · 拿舊版當基線、只套 change request
  • Iterate 會產出機器可讀 <!-- CHANGES --> log
Anchor 檔只吃 Presentkit 產出（偵測 <!-- generated-by: presentkit -->）。任意 HTML 被拒，走 Materials。
10 · Step 7

Generate · 花錢前的三道把關

InfoBar
Cost advisory：模式、預估成本區間、token 估算。< 20K 綠 · 20–28K 黃 · ≥ 28K 紅 + 可行動建議
Radio
Output language：Chinese（1 call） / English（1 call） / Both（2 calls · ≈2× cost）。語言決策發生在花錢前
Pre-flight
Confirm dialog：按下 Generate 後彈，顯示 model + cost + token + rate-limit 風險。預設按鈕是 Cancel · mock mode 跳過

Step 6 Outline 是靜態 review gate —— 拼好的 Intake + Anchor 顯示給使用者最後確認 · 無 API 呼叫 · 無成本。

11 · Step 8
Done

四個檔案 · 一套完整產物

  • resume.html · 主產出 · 自足 HTML
  • resume.en.html · Both 模式才生
  • source-notes.md · 輸入可重現
  • generation.log.md · 元資料 + iterate 時帶 CHANGES
三個按鈕：開 HTML、開資料夾、重新開始一份。設計刻意：檔案保留、wizard 重置。
Section IV · BYOK safeguards

信任
當作 feature

use own key · see own spend · reconcile own billing
12 · Mode badge

頂部三色徽章 · 永遠可見

LIVE

琥珀色 · 會花 token

真 API key 已設 · 按 Generate 會扣錢 · 刻意不用 red（會誤解為錯誤）

OFFLINE

板岩灰 · 安全模式

MockClaudeClient · 零 API 呼叫 · 零成本 · 刻意不用 green（會被讀成 "go"）

NO KEY

鋅灰 · 尚未設定

沒 key · Generate 會失敗 · 提示走 Settings 貼 key 或切 OFFLINE

每次關 Settings dialog 都 refresh —— 使用者永遠知道自己正處在哪個模式。

13 · Pre-flight + audit

Generate 前確認 · Generate 後留紀錄

Pre-flight confirm dialog
顯示 model、成本區間、token 估算、rate-limit 風險標記。Cancel 是預設按鈕 —— 使用者要明確同意。Mock mode 完全跳過這個 dialog。
Token estimate 三階段
< 20K 綠 · 20–28K 黃 · ≥ 28K 紅 + 建議（「remove Anchor / trim Materials / skip LinkedIn」）。
API call audit log
每次真呼叫寫一行到 %LOCALAPPDATA%\Presentkit\api-calls.log（tab-separated：timestamp / model / kind / tokens）。Mock 不寫。
Stale-pref protection
每次真 API call 前重讀磁碟 prefs。早期曾 cache，結果使用者切 OFFLINE 後 Generate 還偷打真 API —— 這條直接壓死。

對帳的時候這份 audit log 可以對齊 Anthropic Console billing —— 「我到底花了什麼」永遠 10 秒內能自驗。
