Packaging an AI resume workflow as a Windows app

Presentkit turns a Claude Code skill workflow into a BYOK desktop product — "open app → fill wizard → get an HTML resume". 01 · Problem & Brand position

From "you need Claude Code" to "double-click the exe"

claude-present already has a battle-tested bilingual resume skill — resume-profile. The problem is you have to cross three thresholds to use it: install the Claude Code CLI, understand the .claude/skills/ layout, and hand-edit Markdown. That's three barriers for a non-developer, and mild friction even for a developer.

Three frictions

Friction 1 · CLI
User must install Node, install Claude Code, and sign in (or bind a MAX subscription) before firing the first command.
Friction 2 · Skill structure
Need to know where SKILL.md lives, which references file changes what output, what the output folder layout looks like.
Friction 3 · Markdown editing
Intake data comes in by hand-writing or pasting Markdown — by editing files. No form, no validation, no bilingual toggle button.

Presentkit's answer — not a prompt wrapper, a product

Plenty of "AI resume tools" on the web are a form that forwards a prompt to GPT/Claude; what the user takes on is a behavioural trust leap ("will it store my resume?"). Presentkit goes the other way: a BYOK (bring your own key) desktop app that hands both cost and control back to the user.

Brand position · spending trust as a feature: other tools treat BYOK as a "technical option"; Presentkit treats it as the product's substance — mode badge, pre-flight dialog, audit log all exist to answer "you always know what you're about to spend and why."
02 · Architecture at a glance

Five projects · one data flow

A WinUI 3 unpackaged desktop app, split into 5 .NET projects. The rule is strict: Presentkit.Core cannot touch I/O; it holds pure data records and design presets only. All file / DPAPI / network code sinks into its own project. This lets Core run under pure unit tests and lets the UI be replaced without touching the domain model.

Five key technical decisions

MVVM · partial-prop
Every VM uses public partial string X { get; set; } with [ObservableProperty]. Field-based form emits MVVMTK0045 (not AOT-safe for WinRT marshalling) — the repo rule is zero warnings.
BYOK · DPAPI
API key stored at %LOCALAPPDATA%\Presentkit\secrets\anthropic-api-key.bin, encrypted with Windows DPAPI CurrentUser scope. No admin rights required; the key becomes invalid if the user migrates machines (deliberately).
PDF · PdfPig
LinkedIn PDF text extraction uses PdfPig (MIT), avoiding iTextSharp's AGPL contagion. ContentOrderTextExtractor preserves column / paragraph structure, which keeps the LinkedIn regex parser happy.
Deploy · self-contained
Unpackaged WinUI3 + PublishReadyToRun + WindowsAppSDKSelfContained. Yields a 103 MB ZIP with no .NET runtime install, no MSIX cert, no admin rights.
Mock · airtight
Only ClaudeClientFactory.Create(apiKey, useMock) ever returns a real client. Any call path that bypasses it breaks the offline toggle — so CLAUDE.md lists it as a hard prohibition.
03 · BYOK safeguards layer

Seven guardrails · turning trust into product

The moment the user presses Generate, real money comes off their API key. Every guardrail is designed around "the user always knows what they're about to spend and why". This isn't compliance checkbox stuff; it's the product's substance.

Pre-flight dialog + audit log, taken together, make "did I just spend 5 cents correctly?" a question the user can self-verify in under 10 seconds.
04 · Anchor & iteration mode

From "make a new resume" to "update the old one"

Real-world resume usage is rarely a from-scratch run. It's usually: last run was fine but I need to add a new project, change the headline, drop an old role. The Anchor step exists for this: drop in a prior Presentkit output, pick a mode.

Two modes

Mirror style
Use the prior resume's voice / structure / density as a style reference, but regenerate content from the current Intake. Use when "last time's tone was right, but the job target / content has moved on".
Iterate (diff mode)
Treat the prior resume as the content baseline, applying only the changes you typed into the Change Request box. Produces a new version plus a machine-readable changelog. Use for "add this project / remove this role / change my headline".

Iteration yields a machine-readable changelog

The iterate-mode system prompt explicitly requires Claude to emit an <!-- CHANGES --> comment block at the top of its HTML body. The orchestrator slices it out and lifts it into generation.log.md. Result: "what changed" is discoverable without opening the HTML.

generation.log.md · ## Claude-reported changes - ADDED: Experience section gains "2026-Q2 LLM integration project" - bullet: Built internal LLM routing layer handling 200k req/day - bullet: Reduced model cost by 34% via intelligent caching - MODIFIED: Hero tagline changed from "Backend engineer" to "Backend + AI infra engineer" - REMOVED: 2021 internship entry (older than 5 years, off-target for current role) - UNCHANGED: Education / Skills sections

Recursive bloat guard

The first iterate run serializes Intake + Anchor into source-notes.md. A naïve implementation would dump the whole prior Anchor into the Intake snapshot — and the next iterate would dump that source-notes again — and the one after is N². The orchestrator's CompactAnchorForArtifact swaps the nested anchor's heavy fields (resume.html / source-notes / generation.log) for a one-line "see prior folder" reference, breaking the exponential recursion.

Technical note: this bug was real — early versions hit 40K+ input tokens at the third iterate, over the 30K/min starter-tier cap. The fix also trimmed the iteration system prompt by ~68%, removing repeated context.
05 · Ship-it cadence

Phase 0 → v0.2 rhythm

From the first empty Mica window to tagging v0.2.0, a few traps got crossed. This section isn't a changelog — it's the short list of mines worth warning the next person about.

Figure 2 · Coral nodes are actual traps hit and fixed (not release milestones)

Trap 1 · [ObservableProperty] on fields emits MVVMTK0045

MVVM Toolkit 8.4+ on WinUI 3 / WinRT emits MVVMTK0045 for field-based [ObservableProperty] — CsWinRT marshalling isn't AOT-safe. Fix: switch to partial-property form. Repo rule: zero-warning builds.

Trap 2 · Click="Handler" inside a DataTemplate doesn't reach the outer UserControl

Once a DataTemplate carries x:DataType (required for compile-time x:Bind), a Click="Handler" inside looks for the handler on the item type (e.g. JobExperienceViewModel), not the outer UserControl. Fix: expose a RemoveCommand on the item, wire it via x:Bind.

Trap 3 · dotnet publish doesn't copy Presentkit.App.pri

Unpackaged WinUI 3's dotnet publish doesn't copy the app-level PRI file from bin/Release/.../win-x64/ into the publish directory. Symptom: double-clicking the unzipped exe silently crashes, no window. Fix: scripts/publish.ps1 force-copies the PRI after dotnet publish before zipping.

Trap 4 · Language fidelity drift

LinkedIn PDFs are usually English; Intake is usually Traditional Chinese. Without explicit steering, Claude follows the dominant input language — so picking "Chinese" could yield English output. Fix: three-way reinforcement — put ## Target output language at the very top of the user message, add a CRITICAL block to the system prompt, and reinforce at the user message's tail.

Other traps not getting their own heading: Claude occasionally wraps HTML in ```html ... ``` markdown code fences (fixed with a system-prompt prohibition + defensive sanitizer); Mirror-anchor mode sometimes copies the anchor's <footer> into Claude's body, producing two footers in the output (fixed with StripTrailingFooter); the recursive-bloat case already covered above.

Closing

Three takeaways worth carrying

01 · Packaging an AI tool = letting the user predict the cost

However good the tech, a BYOK product's first decisive fight is: can the user predict what pressing Generate will do? Mode badge, pre-flight, audit log together aren't compliance; they're the product.

02 · A null-value KPI is still a KPI

"0 installer · 0 admin · 0 CDN" — a KPI for what you don't do — signals trust louder than a feature count. Local-first isn't a slogan; it's a measurable promise.

03 · Treat iteration as a first-class citizen

A resume isn't a one-shot artefact — it's an accumulation of versions. Anchor + <!-- CHANGES --> make "update an old one" as natural as "generate a new one". The system acknowledges, from day one, that humans don't get it right in one pass.
