01 · Problem & Brand position

把「要會 Claude Code」變成「雙擊 exe」

claude-present 有一套已驗證的雙語履歷 skill —— resume-profile。問題是用它要先跨三道門檻：裝 Claude Code CLI、讀懂 .claude/skills/ 結構、手動編輯 markdown。這對非開發者是三道拒馬，對開發者也是小鬱悶。

三道摩擦

Friction 1 · CLI
使用者要裝 Node、裝 Claude Code、登入帳號（或綁 MAX 訂閱）才能按下第一個指令。
Friction 2 · Skill 結構
要知道 SKILL.md 放哪裡、touch 哪個 references 檔會影響什麼產出、output 資料夾結構長怎樣。
Friction 3 · Markdown 編輯
Intake 資料靠手寫或貼上 markdown、靠改檔案來改內容 —— 沒表單、沒驗證、沒雙語切換按鈕。

Presentkit 的答案 —— 不是 prompt wrapper，是產品

市面上很多「AI 履歷工具」是 web 表單直接丟 prompt 給 GPT/Claude，使用者承擔的是行為層面的信任：「它會不會偷存我的履歷？」Presentkit 走另一條：BYOK（bring your own key）桌面 app，把成本和控制權都交還給使用者。

品牌定位 · spending trust as a feature：其他工具把 BYOK 當成「技術選項」；Presentkit 把它當成產品的主體 —— mode badge、pre-flight 對話、audit log 都是圍繞「使用者永遠知道自己花了什麼、為什麼」展開。
02 · Architecture at a glance

五個 project · 一條 data flow

WinUI 3 unpackaged 桌面 app，切成 5 個 .NET project。規矩很硬：Presentkit.Core 不能碰 I/O，只能放純資料 record 和 design preset；所有檔案 / DPAPI / 網路都下沉到對應的專屬 project。這樣 Core 可以用純單元測試驗，UI 可以換不用動領域模型。

Data flow · 一次 Generate 做什麼

五個關鍵技術決策

MVVM · partial-prop
全部 VM 用 public partial string X { get; set; } + [ObservableProperty]。欄位形式會觸發 MVVMTK0045（non-AOT-safe WinRT marshalling），專案規矩是零警告。
BYOK · DPAPI
API key 存 %LOCALAPPDATA%\Presentkit\secrets\anthropic-api-key.bin，用 Windows DPAPI CurrentUser scope 加密。免管理員權限、使用者換機即失效（刻意）。
PDF · PdfPig
LinkedIn PDF 文字抽取走 PdfPig（MIT），避開 iTextSharp 的 AGPL 授權感染。用 ContentOrderTextExtractor 保 column / paragraph 結構，給 LinkedIn parser regex 好過日子。
Deploy · self-contained
Unpackaged WinUI3 + PublishReadyToRun + WindowsAppSDKSelfContained。產出 103 MB ZIP，免安裝 .NET runtime、免 MSIX cert、免管理員權限。
Mock · airtight
只有 ClaudeClientFactory.Create(apiKey, useMock) 是取得真 client 的路。任何繞過它的 call path 都會讓 offline toggle 失靈 —— 所以 CLAUDE.md 明文列為禁止項。
03 · BYOK safeguards layer

七條護欄 · 把信任做成產品

使用者按 Generate 的那一刻，他的 API key 實際扣錢。每一條護欄的設計原則都是「讓使用者永遠知道自己在花什麼、為什麼」。這不是合規勾選，是產品主體。

Safeguard 做的事 實作
Mode badge
LIVE OFFLINE NO KEY		 頂部 wordmark 旁永遠一個彩色徽章。LIVE 是琥珀色（警示但不是 red alert）、OFFLINE 是板岩灰、NO KEY 是鋅灰。每次關 Settings dialog 會自動 refresh。 MainWindow.
RefreshModeBadge()
Pre-flight confirm Generate 按下去之前跳對話，顯示 model、預估成本區間、預估 token 數、rate-limit 風險。預設按鈕是 Cancel —— 使用者必須明確點 "Generate (spend tokens)" 才動。Mock mode 跳過這個 dialog。 GenerateView.
GenerateButton_Click
Token estimate 警報 Generate step 上方的 InfoBar 三階段：< 20K 綠燈、20K–28K 黃燈、≥ 28K 紅燈 + 可行動建議（"remove Anchor / trim Materials / skip LinkedIn"）。 PromptAssembler.
EstimateInputTokens
API call audit log 每次真實 Anthropic 呼叫（stream / ping / ping-failed）都寫一行 tab-separated 到 %LOCALAPPDATA%\Presentkit\api-calls.log。Mock 不寫。使用者可拿這份對帳 Anthropic Console 的 billing。 ApiCallAuditLog
.Record
Stale-pref protection 每次真 API 呼叫前都從磁碟重讀 prefs。早期版本曾 cache，結果使用者在 Settings 切 offline 後、Generate 還是偷偷打了真 API —— 這條護欄直接壓死這類 regression。 GenerateViewModel.
RunAsync
Rate-limit pacing 雙語 + 大 prompt 時，若估算 2 × per-call > 25K，自動在兩通之間塞 silent 60s pause（對使用者顯示 "Processing..."，不暴露 rate-limit 機制）。 GenerateViewModel.
BilingualPause
429 retry Anthropic 回 rate_limit_error 時自動 retry 一次（60s 後）。第二次還失敗才把錯誤表面化到 Status line。 AnthropicClaude
Client
設計理由：LLM 產品的信任不在於「模型多準」，而在於使用者能不能預測自己按下去會發生什麼。Pre-flight 對話 + audit log 加起來，就是讓「我剛剛花了 5 美分對嗎？」這個問題每次都能在 10 秒內自己驗證。
04 · Anchor & iteration mode

從「產一份新履歷」到「改一份舊履歷」

真實履歷使用情境很少是從零產。更常是：上次的版本 OK 但要加一個新專案、要改 headline、要移掉一段太舊的經歷。Anchor step 就是為這個設計的 —— 把之前的 Presentkit 產出檔案拖進來，選一個模式。

兩個模式

Mirror style
用上次的 voice / 結構 / 密度作為 style reference，但內容從當次 Intake 重新生。適合「上次口吻對了，但工作換了 / 履歷改方向」。
Iterate (diff mode)
把上次履歷當內容基線，只應用你在 Change Request 輸入框裡寫的變更。產出新版 + 機器可讀 changelog。適合「加專案 / 移經歷 / 改 headline」。

Iteration 產出一份機器可讀 changelog

Iterate 模式 system prompt 明文要求 Claude 在回傳 HTML 最前面塞一個 <!-- CHANGES --> comment block，Orchestrator 再把它切出來塞進 generation.log.md。這樣「改了什麼」可以在不打開 HTML 的情況下看到。

generation.log.md · ## Claude-reported changes - ADDED: Experience 段新增「2026-Q2 LLM 整合專案」 - bullet: Built internal LLM routing layer handling 200k req/day - bullet: Reduced model cost by 34% via intelligent caching - MODIFIED: Hero tagline 從「Backend engineer」改為「Backend + AI infra engineer」 - REMOVED: 2021 的實習經歷（已超過 5 年且與目標職缺無關） - UNCHANGED: Education / Skills 段

遞迴 bloat 的防護

第一次 iterate 時 source-notes.md 會序列化 Intake + Anchor。如果天真地把整個 prior Anchor 也 dump 進 Intake，下一次 iterate 會把 上次的 source-notes 再 dump 一次，再下一次就 N²。Orchestrator 的 CompactAnchorForArtifact 把嵌套 anchor 的重檔（resume.html / source-notes / generation.log）換成一句「see prior folder」，斷掉遞迴指數增長。

技術細節：這個 bug 是真實踩到的 —— 早期版本 iterate 到第三次時，單次 input tokens 暴增到 40K+ 超過 starter tier 的 30K/min 上限。修完同時 iteration 的 system prompt 也砍掉近 68%，不帶重複上下文。
05 · Ship-it cadence

Phase 0 → v0.2 的節奏

從第一個 Mica 空窗到 tag v0.2.0，中間跨過了一些 trap。這節不是 changelog，是把幾個值得別人少踩的雷挑出來。

Figure 2 · 珊瑚色節點 = 實際踩到並修的 trap（非 release）

Trap 1 · [ObservableProperty] on fields 會 warning MVVMTK0045

MVVM Toolkit 8.4+ 在 WinUI 3 / WinRT 情境下，field-based [ObservableProperty] 會 emit MVVMTK0045（CsWinRT marshalling 不 AOT-safe）。修法：改 partial property 形式。repo rule：零警告 build。

Trap 2 · DataTemplate 裡的 Click="Handler" 綁不到 outer UserControl

一旦 DataTemplate 有 x:DataType（編譯期 x:Bind 需要），其中的 Click="Handler" 會去 item type（JobExperienceViewModel）上找，而不是外層 UserControl。修法：item 身上掛個 RemoveCommand，靠 x:Bind 連上去。

Trap 3 · dotnet publish 不會帶走 Presentkit.App.pri

Unpackaged WinUI 3 的 dotnet publish 不把 app-level PRI 檔從 bin/Release/.../win-x64/ 複製到 publish 目錄。症狀：ZIP 解開 double-click 後 silently crash，無 window。修法：scripts/publish.ps1 收完 dotnet publish 後 force-copy PRI 進去再壓 ZIP。

Trap 4 · Language fidelity 漂移

LinkedIn PDF 多半英文，Intake 多半繁體中文。Claude 在沒強制 steering 的情況下會跟著 dominant input language 跑 —— 選了 Chinese 結果出英文。修法：三層保險 —— 把 ## Target output language 放 user message 最前、system prompt 特別 CRITICAL 段、user message 最後再 reminder 一次。

其他沒單獨列的：Claude 回傳 ```html ... ``` markdown code fence（修法：system prompt 明禁 + 防禦性 sanitizer）· Mirror anchor 模式在 Claude body 末端複製 anchor 的 footer 導致輸出有兩個 footer（修法：StripTrailingFooter）· 遞迴 bloat（前一節已談）。

Closing

三個可以帶走的 takeaway

01 · 包裝 AI 工具 = 讓使用者能預測花費

技術再好，BYOK 產品的第一個決勝點是使用者每次按下 Generate 能不能預測結果。Mode badge、pre-flight、audit log 加起來不是合規項目，是產品主體。

02 · 空值 KPI 也是 KPI

0 installer · 0 admin · 0 CDN 這種「沒做什麼」的特性，對使用者的信任比「我做了多少」更有說服力。local-first 不是口號，是可以量化的承諾。

03 · 把 iteration 做成一等公民

履歷不是一次性產物，是 N 個版本的累積。Anchor + <!-- CHANGES --> 讓「改一份舊的」跟「產一份新的」一樣自然 —— 系統設計之初就承認人不會一次對。

v0.2.1 / v0.3 候選（不承諾順序）

  • Draft persistence 擴及 Materials / Anchor / iteration 欄位（v0.2 只存 Intake + Design）
  • Outline step 預覽 Materials / Anchor / change request，不只 Intake
  • Anthropic usage 欄位串到 generation.log.md（目前只有粗估 char count）
  • Presentkit.App.pri copy 改寫成 MSBuild target，讓直接 dotnet publish 也能跑
  • Code signing + ARM64 build · 視使用者群擴大情況決定
