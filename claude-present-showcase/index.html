01 · Problem v2

v1 處理漂移 —— v2 要處理的是 governance

v1 的 claude-present 用 design system + gold standards + iteration logs 解決了 AI 內容「每次重新發明」的漂移問題。但產內容的系統一旦 live，會開始犯**當下看不見、過半年才後悔**的錯。

成熟的系統不是永遠不犯錯，而是能**察覺錯 → 糾正錯 → 把教訓編回架構 → 確保同類錯不再發生**。

這份 v2 不再推銷「我設計了一套系統」這個靜態命題。它展示的是一件活的、持續發生的事：**這套系統在 live 之後如何自我演進**。以下三個 exhibit 是這 session 中真實發生的案例。

02 · Foundation

v1 的基礎仍在，但已不是主角

v1 建立三個核心原則，v2 預設你已理解這些：

  • Design system 作為視覺風格單一真實來源（tokens.json + patterns.md + components/）
  • Skills 必讀 gold standard、禁止空白紙生成（Progressive Disclosure 三層載入）
  • Outputs 附三件組（主產物 + source-notes + generation-log），作為系統自我對齊訊號

這是基礎。v2 要講：**當這個基礎 live 起來、開始犯錯、被糾正、學到東西之後，發生了什麼？**

03 · Evolution Exhibit A · Governance

系統察覺錯誤 → 糾正 → 編碼進架構

2026-04-18：AI 為履歷把「Information and Telecommunications Engineering」誤譯為「資訊傳播工程」。正確應是「電腦與通訊工程」（銘傳大學官方系網為準）。

使用者 review 時 catch 到。當下修正 resume.html 的譯名 —— 但 claude-present 還做了**第二件事**：把教訓編碼進 resume-profile/references/anti-patterns.md 的第 5 條鐵則。

新鐵則內容：「AI 不得自行翻譯學校 / 系所 / 公司 / 職位 / 證照的官方名稱」—— 必須要求 user 提供官方名稱或 URL 為準。

結果：下次 Claude 讀到 SKILL.md 會看到這條規則寫在最嚴重的鐵則區。同類錯**永不再犯**。

Governance
「只修當下 output 的叫維護；把教訓編進規則的叫 governance。」claude-present 做第二件事。
04 · Evolution Exhibit B · Self-publish

系統開始管理自己的對外表達

v1 時發布是人工 —— 把 HTML 複製到 public repo、手動更新 landing、兩個 repo 各自 commit。v2 把整件事 skill 化。

新 skill promote-to-showcase 一條管線做完四件事：

  • 選取：user 必須明確指名（絕不自動 promote all）
  • Privacy 過濾：移除 .private-only 元素、strip internal sync-marker comments
  • Landing 索引重生：依 WORK_CARDS_START/END 等 markers 自動重寫索引
  • 兩 repo commit + push：source 保持 immutable、destination 自動部署

.private-only 是核心設計：**同一份 HTML 服務兩種 audience**。寄 PDF 給 hiring manager 時 email 可見；公開 URL 時 email 自動被過濾。

Design
Private / public 邊界不靠人的記憶、也不靠 code review 盯 —— 靠 marker-driven pipeline 在 promote 的當下自動執行。
05 · Evolution Exhibit C · Identity

處理最敏感的輸出：個人身份

Resume / LinkedIn 是所有輸出類型裡信任成本最高的 —— 一個錯字、一個誇大、一個過時稱號，都可能葬送機會。系統走同一條管線處理，沒有 special case。

inputs/resume/ 目錄收 LinkedIn snapshot PDF、career source data、2022 年舊履歷。resume-profile skill 在 session 中被擴充支援 LinkedIn refresh / rebrand 兩種延伸輸出，以及 interview-material.md 保留「不上履歷但面試要用」的素材。

.private-only marker 在 email 欄位；promote-to-showcase 自動過濾。產出的 Rebrand guide 在「延後執行」情境下保留完整 context，讓未來回來接續工作不需要重跑討論。

Stress test
能處理個人身份輸出的系統，能處理任何類型的輸出。Resume 是信任成本的上限。過了這關，其他類型就是子集。

Takeaway

成熟度不是 output，是 governance loops

AI 內容系統的成熟度，定義不是「它能產什麼」，而是「它有什麼 governance loops」。Drift 是低標；能**自我糾正、自我發布、自我演進**，才是門檻。

每個系統都從「我設計了 X」開始。但長期價值在於：它是否能從自己的錯誤中學、管理自己的對外表達、隨需求演進而不失結構。

claude-present 的 v2 不是宣傳我設計了什麼。它展示的是：**這三件事在這套系統中真的發生了。**
