01 · Problem

AI 生成的最大敵人，是「每次重新發明」

直接丟一句「幫我寫報告」給 ChatGPT / Claude，結果每次都不一樣：配色不同、結構不同、語氣不同。 風格漂移讓 AI 產出難以累積為資產，更別說建立個人識別。

問題不在 AI 不夠強，而在**沒有結構**限制 AI 的自由度。 每次對話都是空白紙，AI 會重新發明輪子 —— 選配色、選字型、選章節順序、選語氣 —— 即使你心裡知道「我要的是跟上次一樣的那種」，沒有檔案化的規則，AI 會重來一次。

02 · Design

Design system 是風格的單一真實來源

tokens.json、patterns.md、components/ 放在根目錄 design-system/， 所有 skill 共用。每個 skill 只決定自己的結構、語氣、章節邏輯，不決定配色字型。

選 coral 當 accent 色不是隨手挑的，是在 amber / coral / lime 三者間做過 WCAG 對比度檢驗 + Tech Bro 審美評估的決策 —— 理由寫在 tokens.json 頂部的 _meta.accent_rationale 欄位。 修配色只改一個地方，所有 skill 產出同步變更。

Core rule
「修改 tokens.json 會影響所有 skill 產出」—— 這句話明文寫在檔頭，讓未來的自己不會不小心動它。
03 · Design

Skills 必須讀 gold standard，不准空白紙生成

每個 skill 的 SKILL.md 強制每次產出前讀至少一份 examples/ 的範例檔。 憑空生成 = 品質掉。讀範例 = 有錨點。

搭配 Progressive Disclosure 三層載入：SKILL.md 的 frontmatter 負責觸發匹配、body 負責主流程、 references / examples 按需讀取。SKILL.md 本身控制在 500 行內，不把所有規則塞進主 prompt 稀釋指令。

Iteration
滿意的產出複製到 examples/ 作為 gold standard；不滿意的具體問題寫進 anti-patterns.md。 每次產出都在微調兩端。
04 · Design

Outputs 本身是迭代訊號

每次產出都附三件組：主產物（index.html）+ source-notes.md（原始輸入逐字保留） + generation-log.md（決策紀錄、採用哪個範例、跳過什麼、避開哪些反模式）。

這不是報告附帶的 metadata，是**未來進化的訓練資料**。 日後 Claude 產出品質若漂移，回去看同類 output 的 generation-log 就能找回當初的判斷依據； 若要調整 skill 行為，看一堆 source-notes 就能發現「使用者其實常給 X 類輸入，SKILL.md 應該為這類輸入設計」。

Takeaway

好的 AI 工具，是「收斂 AI」的工具

生成能力已經是 commodity —— 大家手上都有 GPT / Claude。 產出品質的差距在於：**有沒有結構性地限制 AI 的自由度**。 Design system 限制視覺、anti-patterns 限制內容、examples 限制結構。限制越具體，產出越穩定、越像「你」。

claude-present 的核心不是三個 skill，而是**一套讓 AI 不斷自我對齊的機制**。 每次產出都是對系統的小幅驗證；每個 anti-pattern 都是對未來的 precaution。 這是 AI 內容生成系統的工程化實踐。
