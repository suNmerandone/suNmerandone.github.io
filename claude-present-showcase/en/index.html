01 · Problem v2

v1 fought drift — v2 is about governance

The v1 of claude-present used a design system, gold standards, and iteration logs to solve AI content drift — the "reinvent from scratch every time" problem. But once a content-generating system goes live, it starts making a different class of errors: errors invisible at the moment, regrettable six months later.

Mature systems aren't those that never err. They're those that can notice the error, correct it, encode the lesson back into the architecture, and ensure the same failure class cannot recur.

This v2 doesn't pitch "I designed a system" as a static claim. It shows something living: how this system actually evolved itself after going live. Three exhibits below — all of them happened for real in this session.

02 · Foundation

v1's foundation is still here — just no longer the headline

v1 established three core principles. v2 assumes you've internalized them:

  • Design system as single source of visual truth (tokens.json + patterns.md + components/)
  • Skills must read a gold standard — no blank-canvas generation (three-layer progressive disclosure)
  • Outputs ship three artifacts (deliverable + source-notes + generation-log), turning every run into a self-alignment signal

That's the foundation. v2 asks: what happened once this foundation went live, started making mistakes, was corrected, and learned from them?

03 · Evolution Exhibit A · Governance

Notice → correct → encode into architecture

2026-04-18: While producing a resume, the AI mistranslated "Information and Telecommunications Engineering" as "資訊傳播工程" (Information & Communication Design). The correct term is "電腦與通訊工程" (Computer & Communication Engineering) — verified against the Ming Chuan University department site.

The user caught it during review. The immediate fix was in resume.html. But claude-present did a second thing: it encoded the lesson into resume-profile/references/anti-patterns.md as rule #5, promoted into the top-tier (鐵則) of the anti-pattern list.

The rule: "AI must not translate the official names of schools, departments, companies, titles, or certifications." It must request the official name or source URL from the user — never guess via common-sense translation.

The effect: next time Claude reads SKILL.md, it sees this rule in the top-tier anti-pattern region. The same failure class cannot recur silently.

Governance
"Fixing the current output is maintenance. Encoding the lesson into rules is governance." claude-present does the second.
04 · Evolution Exhibit B · Self-publish

The system now manages its own public expression

In v1, publishing was manual: copy HTML to the public repo, hand-update the landing, commit each repo separately. v2 captures the entire act as a skill.

The new promote-to-showcase skill does four things in one pipeline:

  • Selection: the user must name items explicitly — never auto-promote all
  • Privacy filtering: strips .private-only elements, removes internal sync-marker comments
  • Landing regeneration: rewrites index cards between WORK_CARDS_START/END and RESUME_CARDS_START/END markers
  • Two-repo commit + push: source stays immutable, destination auto-deploys

.private-only is the design crux: one HTML serves two audiences. Emailing a PDF to a hiring manager? Email visible. Public URL? Email filtered. Zero manual intervention.

Design
The private / public boundary is not enforced by human memory or code review — it is enforced at promote-time by a marker-driven pipeline.
05 · Evolution Exhibit C · Identity

Handling the most sensitive output class: personal identity

Resume and LinkedIn carry the highest trust cost of any output type. One typo, one exaggeration, one outdated title can sink an opportunity. The system handles these through the same pipeline — no special case.

inputs/resume/ holds the LinkedIn snapshot PDF, career source data, and a 2022-era resume. The resume-profile skill was extended mid-session to support LinkedIn refresh / rebrand guides and an interview-material.md file preserving "assertive content not suitable for the resume but valuable in the interview."

The .private-only marker wraps the email field. The promote-to-showcase skill filters it automatically. The rebrand guide itself was produced as a deferred-execution artifact — complete context preserved, ready for a future session to resume without re-deriving the strategy.

Stress test
A system that can handle personal-identity outputs can handle any output class. Resumes are the ceiling of trust cost. Passing this test, other types are subsets.

Takeaway

Maturity isn't output — it's governance loops

The maturity of an AI content system is defined not by what it produces, but by what governance loops it contains. Drift is the baseline. Self-correction, self-publishing, and self-evolution — that's the bar.

Every system begins with "I designed X." Long-term value depends on whether it can learn from its own mistakes, manage its own public expression, and evolve with demand while preserving its structure.

v2 of claude-present is not selling what I designed. It shows: all three of these actually happened in this system.
